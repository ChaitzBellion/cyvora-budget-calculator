How to Deploy Your Website on Vercel with a Custom Domain

A complete step-by-step guide to buying a domain, deploying a website, configuring DNS, and going live using GitHub, GoDaddy, and Vercel.

Introduction

If you're a developer, blogger, or someone building your personal brand, deploying your own website with a custom domain is a huge milestone.

In this guide, I'll walk you through my complete journey of launching a website, covering:

  • Domain purchase and registration
  • GitHub integration and version control
  • Vercel deployment and hosting
  • DNS configuration and management
  • SSL/HTTPS setup and security

By the end, you'll have a clear roadmap to launch your own website with a custom domain.

Choosing a Brand Name

Before any technical work, your brand identity matters. Take time to brainstorm names that are:

  • Memorable and easy to spell
  • Relevant to your niche
  • Professional and scalable
  • Unique with low keyword competition

Pro Tip for SEO

Choose a unique domain name with low competition to help your website rank faster in search results. Avoid overly generic names that might be hard to differentiate.

Buying a Domain

Popular domain registrars include:

  • GoDaddy - Wide selection, good support
  • Namecheap - Affordable, user-friendly
  • Google Domains - Simple interface, integrated with Google
  • AWS Route 53 - For advanced users

After purchase, your domain will show an IDLE status, which means it's registered but not yet connected to any hosting.

Setting Up GitHub

Create a Repository

Start by creating a GitHub repository with your website code:

  • index.html - Main page
  • styles.css - Styling
  • script.js - Functionality
  • README.md - Documentation

GitHub serves as your version-controlled source of truth, allowing you to track changes, collaborate, and easily deploy your site.

git init
git add .
git commit -m "Initial commit: website setup"
git push origin main

Deploying with Vercel

Vercel makes deployment incredibly simple:

  1. Go to vercel.com and sign in with GitHub
  2. Click "Import Project"
  3. Select your GitHub repository
  4. Configure build settings (usually auto-detected)
  5. Click "Deploy"

Within seconds, your site is live at project-name.vercel.app. That's the power of modern deployment platforms!

Your site now has a default Vercel domain, making it accessible worldwide while you set up your custom domain.

Connecting Your Custom Domain

To use your custom domain with Vercel:

  1. Go to Vercel Project Settings
  2. Navigate to Domains section
  3. Click "Add Domain"
  4. Enter your custom domain (e.g., example.com)

Initially, you'll see "Invalid Configuration" status. This is normal and expected before DNS is configured.

DNS Configuration (Critical Step)

To make your domain work, you need to update DNS records in your registrar (GoDaddy, Namecheap, etc.).

A Record (Root Domain)

Field Value
Type A
Name @
Value 76.76.21.21

CNAME Record (www subdomain)

Field Value
Type CNAME
Name www
Value cname.vercel-dns.com

What These Records Do

These DNS records tell the internet to route your domain to Vercel's servers. Without them, your domain won't know where to find your website.

SSL & HTTPS Setup

Good news: Vercel automatically provisions an SSL certificate for your domain using Let's Encrypt.

Once DNS propagation completes, your site will be secured with:

  • Green lock icon in browser
  • HTTPS protocol
  • Auto-renewal of certificates

This happens automatically—no additional configuration needed!

Troubleshooting Common Issues

Domain Still Shows "Invalid Configuration"

This means DNS changes haven't propagated yet. DNS propagation can take 5 minutes to 24-48 hours. Check your DNS records are correctly entered in your registrar's dashboard.

Domain Redirects to Wrong Site

Ensure you're using the correct A and CNAME records provided by Vercel. Typos in DNS values are the most common cause.

HTTPS Not Working

Wait for DNS propagation to complete, then refresh. Vercel needs to verify DNS ownership before issuing the SSL certificate.

www and non-www Both Working?

This is actually fine! Both work. You can redirect one to the other in Vercel settings if you prefer.

Next Steps

Congratulations on deploying your website! Here are recommended next steps:

  • Set up Analytics - Use Vercel Analytics or Google Analytics to track visitors
  • Configure Environment Variables - For API keys and sensitive data
  • Enable Auto-deployments - Deploy automatically on GitHub push
  • Set up Email - Add contact forms and email forwarding
  • Optimize Performance - Use Vercel's built-in performance tools
  • SEO Optimization - Add meta tags and structured data
